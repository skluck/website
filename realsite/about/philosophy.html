Programming Philosophy

Unix

Do one thing, and do it well.
Transact and interface with plain text.
Enforce modularity.
Pipes are good.
Advice from Doug Mcilroy.
The Art of Unix Programming by Eric Raymond.

Windows

The WIN API is designed to be abused.
Always statically link. Wasted space is better than DLL hell.
Run everything in user mode. Even drivers.

Programming

Use the right language for the job.
Python is never the right language for the job.
PHP is also never the right language for the job.
Faster is better than slower.
Don't reinvent the wheel, unless you want to.

Web Development and Design

Keep it simple, stupid.
It's not going to render correctly anways.
CGI and nginx do not mix. Just look at this website.

Software, Freedom, and Privacy

Software should conform to the design philosophy of the system it executes on.
Open Source is better than Closed Source, but intellectual property is important.
"Free Software" should be free as in free beer, and free as in freedom.
Open Source should be permissive - it is open, isn't it?
The GPL isn't permissive. The MIT License is (with caveats).
Software should respect the privacy of its users. That includes websites.
