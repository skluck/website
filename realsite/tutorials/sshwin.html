Instructions for using the Secure Shell (SSH) on Windows

What You'll Need

  • PuTTY - The simple and lightweight terminal emulator and SSH/Telnet client.
  • A valid username and password for the server you're connecting to.

Instructions

First, open up PuTTY. You should be greeted by an interface that looks something like this:

Enter the IP address or hostname of the server you want to connect to.
Make sure that the "Connection type" matches your connection.
Once you have entered this information, click "Open". If this is your first time connection, the following should pop up:

This is just a warning explaining the dangers of connecting to a previously unknown server.
By clicking "yes," you tell PuTTY to cache your connection's RSA2 key for later use.
Press "yes" and you should arrive at this screen:

This is the login prompt, or the last step between you and actual access to the system.
First, you must enter your username. Press return (enter), and you should be prompted for a password:

Enter your password. Don't be afraid when you don't see any text appear on the screen.
When dealing with passwords and other secure data, the terminal standard is to hide input completely.
Once you've entered your password, press return. Congratulations! You're now SSHed into a server!

Now What?

Now that you're actually connected to the server via SSH, you can go about your business.
If you're completely new to terminal interaction, here are a few useful key combinations:

  • ls - list the contents of a directory
  • cd [directory] - change the current directory (the one you're in) to the one specified
  • mkdir [name] - make a new directory with the given name
  • touch [name] - make a new file with the given name
  • Ctrl-C - interrupt the current program or process (this is useful for stopping runaway processes)
  • Ctrl-D - kill the current program or process (a more powerful Ctrl-C, use sparingly)
  • who - see who else is online (i.e. connected to the server)
  • talk [user] - message a user through talkd (the talk daemon). Very useful for asking for advice or help.
  • wall [message] - send out a system-wide broadcast of a message.
Furthermore, because the SSH environment you're now working in is very likely a Unix or Unix-like one, you can now use the `ssh' command to nest into even more SSH sessions!
For more information on this, see the page on SSH instructions for *nix.
