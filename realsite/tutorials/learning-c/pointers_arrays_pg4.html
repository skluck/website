Page 4: Pointers and Arrays

Fun fact: array notation in C is actually just syntactic sugar for pointer notation.

Pointers

As mentioned in the last section, a pointer in C holds the address in RAM of a specific variable. For example: 


		int i = 5;
		int *i_pointer = &i;
In this example, i_pointer now contains the location in memory of i, or 5. The ampersand is used to express the "address of" a variable.
Now, pointers may not seem very useful at first. However, they are the only way to pass by reference in C.
This can be incredibly useful in input functions like scanf: 

		int a = 0, b = 0, c = 0;
		int *ap = &a, *bp = &b, *cp = &c;

		scanf("%d %d %d", &a, &b, &c);
		scanf("%d %d %d", ap, bp, cp);
It's very important to understand that the two scanf calls in the code above do the exact same thing. In fact, the second call will overwrite the variables stored by the first call.
The only difference is how the variables are accessed and stored. One uses direct "address of" notation to skip declaring pointer variables, while the other explictly declares a pointer to each variable to be scanned into. These pointers can then be accessed by dereferencing: 

		printf("%d %d %d\n", a, b, c);
		printf("%d %d %d\n", *ap, *bp, *cp);
Again, these two calls do the exact same thing, but through different processes. Because ap, bp, and cp are pointers, they cannot be printed out as decimals (Well, they can, but something weird will happen. Try it out.) They must be dereferenced, which is to say that the data stored they point to must be accessed.
