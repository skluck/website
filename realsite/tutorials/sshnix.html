Instructions for using the Secure Shell (SSH) on Unix and Unix-like OSes

What You'll Need

  • A fully or reasonably POSIX compliant OS. This includes *BSD, Linux, OS X, Solaris, and Cygwin on Windows.
  • OpenSSH or another implementation of SSH. This is almost always already installed. If it isn't, use your package manager to install it.
  • A valid username and password for the server you're connecting to.

Instructions

First, open up your terminal emulator. On OS X, this is an application called "Terminal."
On other Unix-like OSes, this can be xterm, rxvt, or any other user-installer terminal emulator.
You should be greeted by something similar to this:

The terminal has a number of incredibly useful functions and utilities, but we'll be focusing on the `ssh' utility, which provides full terminal access to a remote system.
To begin, type `ssh' followed by your username and the remote system's IP or hostname.
The format of this section is exactly like an e-mail address:

Press return, and you should receive some kind of warning regarding the server not being recognized.
This is because new connections do not have the target server's key stored.
Answer yes to this, and your computer will cache a unique key for future connections.
Press return, and you should be prompted for a password:

Enter your password carefully, as most terminals do not echo input in secure fields (like passwords).
Press return.
Congratulations! You're now SSHed into a remote host!

Now What?

Because an SSH session on a remote host is virtually identical to a physical terminal session on a local host, you have access to many, all, or even more of the commands you had on your physical host.
Provided the environment is POSIX or POSIX-like, you should feel right at home.
If you have any questions, don't be afraid to message another user with "talk" or to send out a systemwide request with "wall".
